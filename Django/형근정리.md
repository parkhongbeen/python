```
- 환경세팅
    - 가상환경 만들기    ( $ pyenv virtualenv <version> <name>
    - 가상환경 설정       ( $ pyenv local <name> )
    - 장고 설치             ( $ pip install django~=2.2.8 ) ( 2.2.8 버전으로 설치)
    - 프로젝트 시작       ( $ django-admin startproject <name> . ( .은 현재위치를 나타냄)
- 인터프리터 설정
    - File - setting - project~ - project interpreter
    - /Users/<home name>/.pyenv/versions/<env name>/bin/python
    
- 데이터베이스 연동
    - python manage.py migrate          ( 기본으로 생성됐던 11개 연결됨 )
    - python manage.py startapp <app_name>    ( 프로젝트 내부의 별도 애플리케이션 생성)
    - 애플리케이션의 models.py에서 model 설정, setting.py의 INSTALLED_APPS 에 <app_name> 추가
    - python manage.py makemigrations <app_name> // python manage.py migrate <app_name>
        * 데이터베이스 생성 및 연동으로 12개가 됨.
- 관리자 페이지 생성
    - python manage.py createsuperuser      ( 계정 생성 )
        * localhost:8000/admin에 접속 가능하나 아직 POST를 생성하는것이 존재하지 않음
    - admin.py에 admin.site.register(Post) 추가하면 POST를 생성할 수 있는 버튼이 생성 됨.
- shell-plus 환경 설정
    - pip install django-extensions // setting.py 의 INSTALLED_APPS에 django_extentions 추가
        * python manage.py shell_plus를 사용할 수 있게 됨
    - pip install notebook을 설치하면 python manage.py shell_plus --notebook을 통해
      jupyter notebook에서 shell_plus의 기능을 사용할 수 있음.
- html 페이지 생성
    * bootstrap을 연동하기 위한 static 폴더 생성 후 연동
        - STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]
    - post-list.html
        - render를 통해 html을 주기 위해 html 문서들을 저장하고 있는 templates 폴더를 만듦.
        - settings.py에서 폴더 주소를 연동해줌
    - post-detail.html / post-add.html 생성 및 각각 html에 대해 url연결과 views.py에 함수 생성
    - base.html 을 이용해서 묶어주기
        - base파일에서 {% block <content name> %} {% endblock %} 으로 만들 수 있음.
        - {% extends 'base.html' %}
- html 문서의 주소를 동적으로 설정하기
    - urls.py에서 name 값을 주어서 html문서에 적용하면 동적(하나만 바뀌어도 모두 적용)으로 사용할 수 있음.
    - detail의 경우 페이지 마다 값이 다르므로 <(data type):(name)>형태로 주소에 동적인 값을 적용할 수 있음
        * 동적인 값을 주었다면 views.py에서도 값을 인자로 받아서 활용해야 함
- post-add
    - 전송할때와 입력페이지가 다르므로 method가 GET인 경우와 POST인 경우를 나눠서 설계해야함.
```
